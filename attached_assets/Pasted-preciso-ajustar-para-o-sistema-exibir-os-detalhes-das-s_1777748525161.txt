preciso ajustar para o sistema exibir os detalhes das solicitacoes atraves de um menu dropdown na solicitacao:

segue documentação da api para terceiros do voalle:

https://documenter.getpostman.com/view/16282829/UVC3moiN#95b0502e-3209-41c9-932b-6cbc86ba42a5

login:

POST
Autenticação
{{url_do_ERP}}:45700/connect/token
HEADERS
Content-Type
application/x-www-form-urlencoded
Body
urlencoded
grant_type
client_credentials
Valor Fixo
scope
syngw
Valor Fixo
client_id
Token Client_id
Informado pelo cliente - “Suíte / Configurações / Usuários”, campo "Client Id" do usuário marcado como Integrador
client_secret
Token Client_secret
Informado pelo cliente - “Suíte / Configurações / Usuários”, campo "Client Secret" do usuário marcado como Integrador
syndata
Token SynData
Informado pelo cliente - “Suíte / Configurações / Parâmetros” menu “Integração/Mapa” ou solicitado a Voalle via protocolo

Example Request:

curl --location 'https://erp.cliente.com.br:45700/connect/token' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'grant_type=client_credentials' \
--data-urlencode 'scope=syngw' \
--data-urlencode 'client_id=Token Client_id' \
--data-urlencode 'client_secret=Token Client_secret' \
--data-urlencode 'syndata=Token SynData'

Example Response:

{
  "access_token": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "expires_in": 3600,
  "refresh_token": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "token_type": "Bearer"
}

POST
Busca detalhes da solicitação
{{url}}:45715/external/integrations/thirdparty/projects/getsolicitationdata?assignmentId={{assignmentId}}&protocol={{protocol}}
Busca as informações gerais de uma determinada solicitação.
	•	Caso sejam informados ambos os campos na pesquisa, a mesma é realizada levando em consideração o campo "assignmentId".
HEADERS
Authorization
Bearer {{access_token}}
Token obtido no método de autenticação
PARAMS
assignmentId
{{assignmentId}}
ID do protocolo
protocol
{{protocol}}
Número do protocolo

Example Request:

curl --location --request POST 'httpd://erp.cliente.com.br:45715/external/integrations/thirdparty/projects/getsolicitationdata?protocol=123' \
--header 'Authorization: Bearer' \
--data ''
